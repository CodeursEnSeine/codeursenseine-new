description: "Un bon développement se termine toujours par une revue de code. Même si cette étape peut paraître anodine, en fait, elle peut induire beaucoup de choses sur le bon déroulement d'un projet. En effet, une revue de code, paradoxalement, ne concerne pas que le code en lui-même. La revue de code est une activité qui peut avoir des répercussions si l'on ne l'aborde pas de la bonne manière.
Je vous propose de faire le point sur la revue de code. De voir les impacts dans un projet au-delà du code, ainsi que de vous donner les clefs pour réaliser une bonne PR review."
